# 설계: 카테고리 메뉴(섹션 랜딩)

- 날짜: 2026-06-26
- 상태: 승인됨 (사용자 확정)
- 관련: `2026-06-25-krds-website-builder-design.md`(전체), `SitemapTree.tsx`, `lib/sitemap.ts`

## 배경 / 문제

현재 모델은 **사이트맵 노드 1개 = 페이지 1개(1:1 불변식)** 라서, 하위를 가진 부모 메뉴도
자기 페이지를 갖는다. 정부/KRDS 사이트의 1뎁스 메뉴(예: "회사소개")는 보통 **자체 콘텐츠 없이
하위로 펼쳐지거나 첫 하위 페이지로 이동**하는 "카테고리"다. 부모 페이지에 내용을 안 쓰면 빈 페이지가
보여 어색하다.

## 결정 사항 (브레인스토밍 확정)

1. 처리 방식 = **섹션 랜딩(첫 하위로 이동)**. 노드를 "카테고리"로 켜면 자체 콘텐츠 대신 첫 하위 페이지로 보낸다.
2. 켜고 끄기 = 사이트맵 트리 행의 **카테고리 토글 아이콘 버튼**(노드 단위).
3. 에디터에서 카테고리 노드 **제목 클릭 → 첫 하위 콘텐츠 페이지로 자동 이동**(A안). 카테고리 자체 편집화면 없음.
4. 토글 버튼은 **하위가 있는 노드에만** 노출(B 아님). 하위가 없어지면 플래그는 남되 **자기 페이지로 폴백**.

## 모델

- `SitemapNode`에 `isCategory?: boolean` 추가(`src/lib/types.ts`).
  - **1:1 불변식 유지**: 카테고리 노드도 페이지를 그대로 보유한다(직접 안 보이고 첫 하위로 보냄).
  - optional: 옛 영속 데이터엔 없으므로 `=== true`로만 카테고리 판단.

## 순수 헬퍼 (`src/lib/sitemap.ts`)

```
resolveTargetPageId(sitemap: SitemapNode[], nodeId: string): string
```

- 노드를 찾지 못하면 입력 nodeId를 그대로 반환(방어).
- 노드가 **카테고리가 아니면** 그 노드의 `pageId` 반환.
- 카테고리면 **하위를 전위(pre-order) 탐색**해 처음 만나는 **비-카테고리** 노드의 `pageId` 반환
  (카테고리 체인은 계속 내려가 실제 콘텐츠 페이지에 도착).
- 카테고리인데 비-카테고리 하위가 하나도 없으면(또는 하위 없음) **자기 `pageId`로 폴백**.

전위 탐색 순서 = `children`을 순서대로 보되 각 자식에 대해 "자식 자신 → 그 자식의 하위" 순으로 본다
(첫 자식이 카테고리면 그 자식의 하위로 먼저 내려간다).

## store 액션

- `setNodeCategory(id: string, isCategory: boolean): void` — 해당 노드의 `isCategory`를 불변 갱신.

## SitemapTree UI

- **카테고리 토글 버튼**(아이콘)을 행 액션에 추가 — **하위가 있는 노드에만** 노출.
  현재 상태를 아이콘으로 표시(켜짐=강조/채움, 꺼짐=외곽선), `aria-pressed`로 상태 전달. `setNodeCategory` 호출.
- **제목 클릭 동작 분기**: `node.isCategory`면 `setActivePage(resolveTargetPageId(sitemap, node.id))`
  → 첫 하위 콘텐츠 페이지로 이동. 일반 노드는 기존대로 `setActivePage(node.pageId)`.
- 카테고리 노드는 시각적으로 구분(제목 옆 작은 "카테고리" 배지/아이콘).
- 드래그·인라인편집·다른 액션은 기존과 동일.

## 데이터 흐름

트리에서 카테고리 토글 → `setNodeCategory` → 리렌더. 카테고리 제목 클릭 →
`resolveTargetPageId` → `setActivePage(첫 하위)`. 캔버스/LNB는 기존 그대로(카테고리의 빈 자체
페이지는 클릭 리다이렉트로 사실상 노출되지 않음).

## 테스트 / 검증

- **순수 `resolveTargetPageId`**(TDD red→green):
  - 비-카테고리 노드 → 자기 pageId.
  - 카테고리 → 첫 하위(비-카테고리)의 pageId.
  - 카테고리 체인(카테고리의 첫 자식도 카테고리) → 더 내려간 첫 콘텐츠 pageId.
  - 하위 없는/비-카테고리 하위 없는 카테고리 → 자기 pageId 폴백.
  - 없는 nodeId → 입력 반환.
- **store `setNodeCategory`**: 토글 반영, 다른 노드 영향 없음.
- **컴포넌트(SitemapTree)**: 토글 버튼이 하위 있는 노드에만 노출 / 카테고리 제목 클릭 시
  activePageId가 첫 하위로 / 토글이 `isCategory`를 켬. (testing-library)

## 범위 밖 (후속/백로그)

- **익스포트(Step 7)**: 카테고리 페이지 = 첫 하위로 HTML 리다이렉트(meta refresh 또는 첫 하위 링크) 배선.
- 브레드크럼/캔버스에서 카테고리 별도 표현(읽기전용 안내 등) — 지금은 클릭 리다이렉트까지만.
- LNB에서 카테고리 링크가 첫 하위 경로를 가리키게 하는 export 정합성 — Step 7과 함께.
